How to use this map

Four view modes change what the canvas emphasizes. Use them after you load a workflow (and optionally a log or What-If run).

1. Topology (default)

  1. Open a workflow map — every task and edge is visible.
  2. Click any task → details appear in the Diagnostics Panel (Logic / lineage / trace / cluster).
  3. Use Fit / search to navigate large maps.
Example: click on Retrieve Approval Records → check its query filters on the Logic tab.

2. Live Trace

  1. Upload a workflow XML and a matching live execution .log (web UI), or generate a map with a log via CLI.
  2. Click Live Trace — teal edges/nodes show what actually ran; the rest dim.
  3. Click a live task → Trace evidence tab shows log lines for that step.
Example: if a production workflow failed mid-run → Live Trace shows which Switch branch was taken before it stopped.

3. What-If

  1. In the web UI (above the map), type a scenario or click chips like Force FALSE — Switch….
  2. Click Simulate — failed/altered/broken tasks glow; off-path tasks dim.
  3. Click What-If on this strip anytime to re-show that overlay (after Clear, glow is gone until you Simulate again).
Example: “What if the operational status is DISP?” → Simulate → purple path + grey starved dependents.

4. Diff

  1. Upload two or more workflow XMLs in one session (web UI).
  2. Open map A, click Diff — green glow = tasks only in A; red = only in the other map.
  3. Use this to spot customization drift between baseline and custom WF versions.
Example: baseline PermanentSaveValidation vs your customized copy → Diff highlights renamed/added tasks.

Handy panel actions

  • Show path to here — spotlight Start → selected task.
  • Copy deep link — share a URL that re-opens this task (and mode) later.
  • Cluster tab — when you click an Iter/Loop container, lists nested tasks.
